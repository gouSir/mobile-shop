## 一、项目背景

### 1.1 项目开发的目的

随着移动互联网的普及和电子商务的迅猛发展，在线购物已经成为人们日常生活中不可或缺的一部分。手机作为最热门的消费电子产品之一，其线上销售市场规模逐年增长。本项目的开发目的主要包括以下几个方面：

**（1）实践 Java Web 核心技术。** 本课程设计旨在将《Java Web 程序设计》课程中所学的理论知识应用于实际项目开发，重点实践 Servlet、JSP、JDBC、Session 管理等核心技术，加深对 B/S 架构和 MVC 设计模式的理解。

**（2）掌握数据库设计与编程。** 通过设计完整的数据库表结构（用户表、商品表、购物车表、订单表、订单明细表），实践 MySQL 数据库的 DDL 与 DML 操作，掌握 JDBC 数据库连接、PreparedStatement 预编译、事务管理等技术。

**（3）完成一个功能完整的电商系统。** 从用户注册登录、商品浏览搜索，到购物车管理、订单提交与管理、后台管理，实现电商平台的核心业务流程闭环，体验一个完整项目的开发过程。

**（4）积累项目调试经验。** 在实际开发中遇到并解决各种问题（依赖缺失、数据库不匹配、编码乱码等），提高独立排查问题和解决问题的能力。

### 1.2 查阅的项目相关文献和资料综述

在项目开发过程中，主要参考了以下文献和资料：

**（1）Java Web 开发教材。** 参考了明日科技编著的《Java Web从入门到精通（第4版）》（清华大学出版社，2021），系统学习了 Servlet 生命周期、JSP 九大内置对象、JSTL 标签库、EL 表达式等内容，为项目开发奠定了理论基础。

**（2）JDBC 数据库编程。** 参考了李刚编著的《轻量级Java EE企业应用实战（第5版）》（电子工业出版社，2020），学习了 JDBC 连接管理、PreparedStatement 的使用、ResultSet 结果集处理以及数据库事务的 ACID 特性。

**（3）Bootstrap 前端框架。** 查阅了 Bootstrap 5 官方文档（getbootstrap.com），学习了栅格布局系统、导航栏组件、卡片组件、轮播组件等的使用方法，用于构建响应式的用户界面。

**（4）Maven 构建工具。** 参考了 Maven 官方文档和网络博客，学习了 pom.xml 的配置方法、依赖管理机制以及 tomcat7-maven-plugin 插件的使用。

## 二、需求分析

### 2.1 系统总体需求

本项目需要开发一个手机在线购物商城系统。系统采用 B/S 架构，用户通过浏览器访问，服务器端使用 Java Web 技术处理请求，MySQL 数据库存储数据。

系统需要支持两类用户角色：**普通用户（user）**和**管理员（admin）**。

### 2.2 普通用户功能需求

**（1）用户注册与登录。** 新用户可以通过注册页面创建账号，填写用户名、密码、邮箱、电话、地址等信息。已注册用户通过用户名和密码登录系统。登录成功后，系统将用户信息保存在 Session 中，页面头部根据登录状态动态显示不同的导航菜单。

**（2）商品浏览与搜索。** 用户可以浏览全部商品列表，支持按品牌（Apple、华为、小米、三星、OPPO、vivo、一加、荣耀等）筛选商品，按分类（旗舰机、拍照手机、游戏手机、性价比）筛选商品，以及通过关键字搜索商品（搜索范围包括商品名称、品牌和描述）。

**（3）商品详情查看。** 点击商品可进入详情页面，查看商品的完整信息，包括名称、品牌、描述、价格、库存、分类等。

**（4）购物车管理。** 登录用户可以将商品加入购物车，在购物车页面修改商品数量或删除商品。同一商品重复添加时会自动合并数量。购物车列表通过 JOIN 查询关联商品信息，实时显示商品名称、品牌、单价和金额小计。

**（5）订单提交与管理。** 用户可以将购物车中的商品提交为订单，填写收货人姓名、电话和地址。系统自动生成唯一的订单编号（格式：MO + 年月日时分秒 + 用户ID），同时扣减商品库存并清空购物车。用户可以在"我的订单"页面查看所有历史订单及其明细。

### 2.3 管理员功能需求

**（1）商品管理。** 管理员可以查看所有商品列表，添加新商品（填写名称、品牌、描述、价格、库存、图片、分类），编辑已有商品信息，以及删除商品。

**（2）订单管理。** 管理员可以查看系统中所有订单，执行发货操作（状态从"待发货"变为"已发货"），完成订单（状态变为"已完成"），以及删除订单。

**（3）权限控制。** 管理员功能入口仅对 admin 角色可见。后台 Servlet 在每次请求处理前都会检查 Session 中用户的角色，非管理员将被重定向到登录页面。

### 2.4 非功能性需求

- **编码统一**：全局使用 UTF-8 编码，通过 EncodingFilter 过滤器统一处理请求与响应的编码。
- **响应式布局**：使用 Bootstrap 5 框架，确保在不同屏幕尺寸下都能正常显示。
- **数据一致性**：订单创建涉及多表操作，使用 JDBC 事务保证原子性。
- **用户体验**：表单提交有非空和格式验证，操作失败有错误提示信息。

## 三、数据库设计

### 3.1 数据库选择

本项目采用 MySQL 8.0 关系型数据库，数据库名称为 `mobile`，字符集采用 `utf8mb4`，支持存储中文和 Emoji 表情符号。

### 3.2 E-R 图（实体关系图）

系统包含五个实体：用户（users）、商品（products）、购物车（cart）、订单（orders）、订单明细（order_items）。

实体间关系如下：
- 一个用户可以拥有多个购物车记录（1:N），每个购物车记录关联一个商品

- 一个商品可以出现在多个购物车记录和多个订单明细中

## 四、详细设计说明

### 4.1 系统架构设计

本项目采用经典的 **MVC（Model-View-Controller）分层架构**：

- **Model（模型层）**：`com.shop.model` 包，包含 User、Product、CartItem、Order、OrderItem 五个实体类，以及 `com.shop.dao` 包中的数据访问层（UserDao、ProductDao、CartDao、OrderDao），负责与数据库交互。
- **View（视图层）**：`webapp/jsp/` 目录下的 JSP 页面，使用 JSTL + EL 表达式进行数据展示，使用 Bootstrap 5 进行页面布局。
- **Controller（控制层）**：`com.shop.servlet` 包中的 7 个 Servlet 类，负责接收 HTTP 请求、调用 DAO 处理业务逻辑、转发或重定向到视图。

此外还有：
- **Filter（过滤器）**：EncodingFilter 统一设置请求和响应的 UTF-8 编码
- **Utility（工具类）**：DBUtil 封装数据库连接的获取与关闭操作

### 4.3 核心技术说明

**（1）JDBC 数据库连接。** 使用 DriverManager 获取 MySQL 连接，在 static 代码块中加载驱动类 `com.mysql.cj.jdbc.Driver`。连接字符串指定了 `useSSL=false`、`serverTimezone=Asia/Shanghai`、`characterEncoding=utf-8` 等参数以确保兼容性。

**（2）SQL 注入防护。** 所有 SQL 操作均使用 PreparedStatement 预编译语句和参数占位符 `?`，杜绝 SQL 注入风险。

**（3）事务管理。** 订单创建涉及订单主表插入、订单明细批量插入、库存扣减、购物车清空等多步操作。使用 `conn.setAutoCommit(false)` 开启事务，所有操作成功后 `conn.commit()`，任何一步失败则 `conn.rollback()` 回滚，保证数据一致性。

**（4）Session 会话管理。** 用户登录成功后将 User 对象存入 Session。页面头部（header.jsp）根据 Session 中是否有 user 来动态渲染导航栏（登录前显示"登录/注册"，登录后显示"购物车/我的订单/用户名/退出"）。管理员额外显示"管理"下拉菜单。

**（5）批量操作。** 订单明细插入使用 `PreparedStatement.addBatch()` 和 `executeBatch()` 进行批量处理，减少数据库往返次数。

### 5.1 数据库连接工具类（DBUtil.java）

```java
public class DBUtil {
    private static final String URL = "jdbc:mysql://localhost:3306/mobile"
        + "?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf-8";
    private static final String USER = "root";
    private static final String PASSWORD = "root";

    static {
        try {
            Class.forName("com.mysql.cj.jdbc.Driver");
        } catch (ClassNotFoundException e) {
            throw new RuntimeException("MySQL驱动加载失败", e);
        }
    }

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(URL, USER, PASSWORD);
    }

    public static void close(Connection conn, Statement stmt, ResultSet rs) {
        try { if (rs != null) rs.close(); } catch (SQLException e) { e.printStackTrace(); }
        try { if (stmt != null) stmt.close(); } catch (SQLException e) { e.printStackTrace(); }
        try { if (conn != null) conn.close(); } catch (SQLException e) { e.printStackTrace(); }
    }
}
```

```java
    String username = req.getParameter("username");
    String password = req.getParameter("password");

    if (username == null || password == null
            || username.trim().isEmpty() || password.trim().isEmpty()) {

    } else {
        req.setAttribute("error", "用户名或密码错误！");

    }
}
```

### 5.4 订单创建——事务管理（OrderDao.java）

```java
public int createOrder(Order order) {
    String sql = "INSERT INTO orders (user_id, order_no, total_amount, status, "
        + "receiver_name, receiver_phone, receiver_address) VALUES (?, ?, ?, ?, ?, ?, ?)";
    Connection conn = null;
    try {
        conn = DBUtil.getConnection();
        conn.setAutoCommit(false);  // 开启事务

        // 1. 插入订单主表
        ps = conn.prepareStatement(sql, Statement.RETURN_GENERATED_KEYS);
        // ...设置参数、执行
        int orderId = rs.getInt(1);  // 获取自增ID

        // 2. 批量插入订单明细
        String itemSql = "INSERT INTO order_items (...) VALUES (?, ?, ?, ?, ?, ?)";
        PreparedStatement itemPs = conn.prepareStatement(itemSql);
        for (OrderItem item : order.getItems()) {
            itemPs.setInt(1, orderId);
            // ...设置参数
            itemPs.addBatch();  // 添加到批处理
        }
        itemPs.executeBatch();  // 批量执行

        conn.commit();  // 提交事务
        return orderId;
    } catch (SQLException e) {
        if (conn != null) conn.rollback();  // 回滚
        e.printStackTrace();
    } finally {
        if (conn != null) conn.setAutoCommit(true);
    }
    return -1;
}
```

```java
public class EncodingFilter implements Filter {
    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
            FilterChain chain) throws IOException, ServletException {
        HttpServletRequest req = (HttpServletRequest) request;
        HttpServletResponse resp = (HttpServletResponse) response;
        req.setCharacterEncoding("UTF-8");
        resp.setContentType("text/html;charset=UTF-8");
        chain.doFilter(req, resp);
    }
```

```java
private void addToCart(HttpServletRequest req, HttpServletResponse resp, User user)
        throws IOException {
    int productId = Integer.parseInt(req.getParameter("productId"));
    int quantity = (req.getParameter("quantity") != null)
                   ? Integer.parseInt(req.getParameter("quantity")) : 1;

    // 检查购物车是否已存在该商品
    CartItem existItem = cartDao.findByUserAndProduct(user.getId(), productId);
    if (existItem != null) {
        cartDao.updateQuantity(existItem.getId(), existItem.getQuantity() + quantity);
    } else {
        CartItem item = new CartItem();
        item.setUserId(user.getId());
        item.setProductId(productId);
        item.setQuantity(quantity);
        cartDao.add(item);
    }
    resp.sendRedirect(req.getContextPath() + "/cart?action=list");
}
```

### 7.2 调试过程中出现的错误及解决方案

在程序运行的过程中，出现了很多的错误。有很复杂的问题，也不乏许多低级的错误。

**问题 1：MySQL 驱动 JAR 包缺失导致 ClassNotFoundException**

*错误现象*：项目启动后，访问首页时页面显示 HTTP 500 错误。查看控制台日志发现 `java.lang.RuntimeException: MySQL驱动加载失败`，根本原因是 `java.lang.ClassNotFoundException: com.mysql.cj.jdbc.Driver`。

*原因分析*：经排查发现 pom.xml 中配置的 MySQL 驱动依赖坐标为 `mysql:mysql-connector-java:8.0.33`。自 MySQL Connector/J 8.0.31 版本起，Maven 坐标已从 `mysql:mysql-connector-java` 变更为 `com.mysql:mysql-connector-j`。旧坐标下版本 8.0.33 在 Maven Central 仓库中返回 404 错误，导致 JAR 包从未被下载，DBUtil 的 static 块中 `Class.forName("com.mysql.cj.jdbc.Driver")` 找不到驱动类。

*解决方法及步骤*：
（1）进入 Maven 本地仓库目录（`~/.m2/repository/mysql/mysql-connector-java/8.0.33/`），确认仅有 .pom 文件而无 .jar 文件；
（2）使用 curl 测试 Maven Central 仓库，确认 `mysql:mysql-connector-java:8.0.33` 返回 404，而 `com.mysql:mysql-connector-j:8.0.33` 返回 200；
（3）将 pom.xml 中的依赖坐标修改为 `<groupId>com.mysql</groupId><artifactId>mysql-connector-j</artifactId>`；
（4）在 IntelliJ IDEA 中刷新 Maven 依赖并重新编译项目；
（5）重启 Tomcat 服务器，验证驱动加载成功。

---

**问题 2：数据库表结构与 Java 代码不匹配导致 SQLException**

*错误现象*：修复上述问题后，登录页面仍报 500 错误。查看后端日志发现 `java.sql.SQLException: Column 'email' not found`，商品列表页则报 `Column 'brand' not found`。

*原因分析*：使用 MySQL 客户端连接数据库后发现，`mobile` 数据库曾被一个图书管理系统项目使用，其表结构与本项目完全不同。users 表缺少 `email` 和 `address` 字段（实际字段为 `real_name`），products 表缺少 `brand` 和 `category` 字段（实际字段为 `category_id` 外键），role 字段使用 ENUM('reader','admin') 而非 VARCHAR。

*解决方法及步骤*：
（1）通过 `DESCRIBE users; DESCRIBE products;` 确认实际表结构；
（2）由于存在旧表的外键约束（borrow_records_ibfk_1 引用 users），需先执行 `SET FOREIGN_KEY_CHECKS = 0` 禁用外键检查；
（3）删除所有旧表（users、products、cart、orders、order_items、books、borrow_records、categories）；
（4）执行项目 database/init.sql 脚本，重新创建所有表并插入初始数据；
（5）验证数据正确性：`SELECT COUNT(*) FROM products;` 返回 15，`SELECT COUNT(*) FROM users;` 返回 2。

---

**问题 3：购物车重复添加同一商品**

*错误现象*：用户多次点击同一商品的"加入购物车"按钮后，购物车中出现多条相同商品的记录，而非合并数量。

*原因分析*：原 CartServlet.addToCart 方法直接调用 cartDao.add() 插入新记录，没有检查该用户购物车中是否已存在同一商品。

*解决方法及步骤*：
（1）在 CartDao 中新增 findByUserAndProduct(int userId, int productId) 方法，查询 cart 表中是否存在 user_id 和 product_id 同时匹配的记录；
（2）在 CartServlet.addToCart 方法中先调用 findByUserAndProduct 检查；
（3）若已存在则调用 updateQuantity 累加数量，若不存在则调用 add 插入新记录。

---

**问题 4：订单创建时的数据一致性问题**

*错误现象*：在测试订单提交时，偶尔出现订单已创建但库存未扣减、或订单明细未插入的情况。

*原因分析*：订单创建涉及四个数据库操作：插入 orders 表、批量插入 order_items 表、更新 products 表的 stock 字段、清空 cart 表。如果其中一步失败而其他步骤已执行，将导致数据不一致。

*解决方法及步骤*：
（1）在 OrderDao.createOrder 方法中使用 JDBC 事务管理；
（2）调用 `conn.setAutoCommit(false)` 开启手动事务；
（3）依次执行订单插入、明细批量插入、库存更新、购物车清空；
（4）所有操作成功后调用 `conn.commit()` 提交；
（5）在 catch 块中调用 `conn.rollback()` 回滚所有更改；
（6）在 finally 块中恢复 `conn.setAutoCommit(true)`。

## 八、课程实训总结

### 8.1 项目中用到的知识点

本项目综合运用了 Java Web 课程中的多项核心技术：

**（1）Servlet 与 HTTP 协议。** 通过继承 HttpServlet 并重写 doGet/doPost 方法处理客户端请求，理解 HTTP 请求-响应模型。使用 HttpServletRequest 获取请求参数，使用 HttpServletResponse 进行页面重定向（sendRedirect）和请求转发（forward）。

**（2）JSP 与视图层技术。** 使用 JSP 脚本（Scriptlet）和表达式（Expression）嵌入 Java 代码，使用 JSTL 核心标签库（c:forEach、c:if、c:choose）进行数据遍历和条件渲染，使用 EL 表达式（${}）简化数据访问。使用 `<%@ include %>` 指令复用公共页面组件（header.jsp、footer.jsp）。

**（3）JDBC 数据库编程。** 使用 DriverManager 管理数据库连接，使用 PreparedStatement 预编译 SQL 防止注入攻击，使用 ResultSet 处理查询结果集，使用 Statement.RETURN_GENERATED_KEYS 获取自增主键。

**（4）事务管理。** 理解事务的 ACID 特性（原子性、一致性、隔离性、持久性），在订单创建场景中使用 JDBC 事务 API（setAutoCommit、commit、rollback）保证多步操作的原子性。

**（5）Session 会话跟踪。** 使用 HttpSession 存储登录用户信息，实现跨请求的用户状态保持。通过 Session 判断用户登录状态和角色权限。

**（6）过滤器（Filter）。** 实现 Filter 接口，在 web.xml 中配置过滤映射，统一处理全站请求的字符编码。

**（7）Maven 项目管理。** 使用 pom.xml 管理项目依赖（Servlet API、JSP API、JSTL、MySQL Connector），配置 tomcat7-maven-plugin 插件实现一键启动。

**（8）MVC 分层架构。** 将系统分为 Model（实体类 + DAO）、View（JSP 页面）、Controller（Servlet）三层，各层职责清晰，降低耦合度。

**（9）前端技术。** 使用 Bootstrap 5 栅格系统进行响应式布局，使用 Bootstrap Icons 图标库美化界面，使用 CDN 加速静态资源加载。

### 8.2 程序有待改进的地方

**（1）密码安全。** 当前用户密码以明文形式存储在数据库中，存在安全隐患。应使用 MD5 加盐（Salt）或 BCrypt 等算法对密码进行哈希存储。

**（2）数据库连接池。** 目前每次数据库操作都通过 DriverManager 新建连接，高并发场景下性能较差。应引入连接池技术（如 Druid、HikariCP 或 C3P0），复用数据库连接。

**（3）前端架构升级。** 当前为服务端渲染的 JSP 架构，页面刷新频繁、交互体验较差。可考虑引入 Vue.js 或 React 等前端框架，实现前后端分离，通过 AJAX/RESTful API 进行数据交互。

**（4）图片上传功能。** 目前商品图片仅存储文件名，需要手动将图片文件放入 images 目录。可增加文件上传功能，通过 Servlet 3.0 的 Part 接口或 Apache Commons FileUpload 实现。

**（5）分页功能。** 当商品数量较多时，列表页应增加分页功能，通过 LIMIT 和 OFFSET 进行数据库分页查询。

**（6）输入验证加强。** 可使用 JavaScript 在客户端进行更完善的表单验证，同时在服务端对所有输入进行校验和过滤。

**（7）日志系统。** 目前使用 e.printStackTrace() 输出异常信息，应引入日志框架（如 Log4j 或 SLF4J），将日志分级输出到文件。

**（8）版本控制。** 项目未使用 Git 进行版本管理，建议初始化 Git 仓库并提交到 GitHub/Gitee，便于代码回溯和团队协作。

### 8.3 收获和建议

**收获：**

通过本次课程实训，我对 Java Web 开发有了从理论到实践的完整认识。从项目的需求分析、数据库设计、系统架构设计，到具体编码实现、调试排错，完整地体验了一个 Web 项目的开发流程。

在技术层面，我深入理解了 Servlet 的生命周期和工作原理，掌握了 JSP + JSTL + EL 表达式的视图层开发技术，学会了 JDBC 数据库编程和事务管理。在架构层面，我理解了 MVC 分层架构的设计思想，明白了为什么要将数据访问、业务逻辑和页面展示分离。

在调试过程中，我遇到了驱动缺失、数据库不匹配、数据一致性等实际问题。这些问题让我认识到，开发中不仅要关注代码逻辑，还要关注运行环境（依赖管理、数据库配置、文件编码等）。每次排错都是对知识体系的加强，也让我学会了如何从错误日志中定位问题的根源。

**建议：**

（1）建议在课程中增加 Spring Boot 框架的教学。Spring Boot 的自动配置和起步依赖可以大幅简化 Java Web 开发的配置工作，MyBatis 或 JPA 可以让数据库操作更加优雅。

（2）建议学习并使用版本控制工具 Git。在项目中每次修改前提交一次版本，可以大胆尝试新功能，出错时随时回退。

（3）建议在学习经典 Servlet/JSP 开发模式的基础上，了解前后端分离的开发模式。RESTful API + SPA（单页应用）已成为现代 Web 开发的主流。

（4）建议多写注释和文档。本项目虽然功能完整，但代码中的注释还可以更加详细，特别是复杂业务逻辑（如订单创建事务）的注释。
